Overview

This lab will provide a basic understanding of using the AWS Toolkit for .NET. It will demonstrate the basic steps required to install Visual Studio Community Edition and the AWS Toolkit for .NET and to run a simple console application in C# using the AWS SDK for .NET.

Topics covered

By the end of this lab you will be able to:

  • Connect to an Amazon EC2 instance running Windows Server
  • Install AWS Toolkit for .NET
  • Configure Visual Studio for AWS.
  • Create an AWS project in Visual Studio
  • Build a Windows console application to create and read Amazon SQS queues and send notification messages with Amazon SNS

Prerequisites

To successfully complete this lab, you should be familiar with Visual Studio for C# and have a basic knowledge of AWS.

Task 1: Connect to your EC2 development instance

In this task, you will connect to your Windows Development EC2 instance.

What is Amazon Elastic Compute Cloud (EC2)? EC2 is a web service that provides resizable compute capacity in the cloud. It's designed to make web-scale cloud computing easier for developers. Amazon EC2 reduces the time required to obtain and boot a new server instance to minutes, allowing you to quickly scale capacity, both up-and-down, as your computing requirements change.

  1. In the AWS Management Console, on the Services menu, choose EC2.

If you see New EC2 Experience at the top-left of your screen, ensure New EC2 Experience is selected. This lab is designed to use the new EC2 Console.

  1. In the left navigation pane, choose Instances.

  2. Select the Development Server instance.

  3. In the Actions menu, choose Connect.

  4. In the Connect To instance window, select RDP client.

  5. Choose Download remote desktop file

Save the file to your computer and make a note of its location.

  1. Open the .rdp file that you downloaded.

  2. In the Remote Desktop Connection dialog box, choose Connect then configure:

  • Password: Paste the value of AdministratorPassword located to the left of these instructions
  • Choose OK
  1. In the Remote Desktop Connection dialog box, choose Yes to continue to your Windows EC2 instance. It will take a moment to display the Windows Server desktop.

Task 2: Install and configure the required software

In this task, you will configure Visual Studio which is used to modify your AWS code. You will also install and configure the AWS Toolkit for Visual Studio.

The AWS Toolkit for Visual Studio is a plug-in for Microsoft Visual Studio that makes it easier for developers to develop, debug, and deploy .NET applications that use Amazon Web Services. With the AWS Toolkit for Visual Studio, you can get started faster and be more productive when building AWS applications. The AWS Toolkit for Visual Studio conveniently includes the AWS SDK for .NET, so you can start building .NET applications on AWS infrastructure services in Visual Studio, including Amazon Simple Queue Service (SQS), Amazon Simple Notification Service (SNS), Amazon S3, Amazon EC2, AWS Elastic Beanstalk, and Amazon DynamoDB.

Configure Visual Studio 2019 Community Edition

  1. Once you are connected to your instance, choose the Windows Start button and select Visual Studio 2019.

  2. At the Visual Studio Welcome page, choose Not now, maybe later at the bottom of the window.

In this lab, you do not need an account.

  1. At the Visual Studio - Start with a familiar environment window, do the following:
  • In Development Settings: choose the dropdown arrow and select Visual C#
  • Choose Start Visual Studio

C# is an elegant and type-safe object-oriented language developed by Microsoft that enables developers to build a variety of secure and robust applications that run on the .NET Framework. You can use C# to create Windows applications, web services, distributed components, client-server applications, database applications, and much more. Visual Studio provides an advanced C# code editor, convenient user interface designers, integrated debugger, and many other tools to simplify development of applications based on the C# language and the .NET Framework.

  1. At the Visual Studio 2019 page, choose Continue without code.

You will now install the AWS Toolkit for .NET which allows you to interact directly with your AWS environment from inside Visual Studio.

Configure the AWS Toolkit

You must configure the AWS Toolkit for .NET to get access to your AWS environment. This allows you to add, edit, and delete components in your AWS environment, such as Amazon EC2 instances, Amazon SQS queues, and Amazon SNS topics from the Visual Studio environment.

  1. Choose the Windows Start button and select Visual Studio 2019.

  2. At the Visual Studio 2019 page, choose Continue without code.

  3. On the View menu, choose AWS Explorer.

  4. In the AWS Explorer for Profile, select Toolkit.

  5. In the AWS Explorer for Region, select the same region that matches the RegionCode to the left of these instructions.

In AWS Explorer, you can set the Region drop-down list to the AWS region you want to browse. When you develop code to run in AWS, always ensure that the region used by your code is the same region you are viewing in the AWS Management Console and in AWS Explorer. If you don't see resources that you expect to be in your account, ensure the correct region is selected.

Use this page: https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html to identify which region to select based on the region code.

You can now use AWS Explorer to navigate through your AWS environment. For example, in AWS Explorer, expand Amazon S3 to see the Amazon S3 buckets associated with this lab account.

  1. Close Microsoft Visual Studio.

Task 3: Open the AWS SDK console application

In this task, you will open a prebuilt Visual Studio solution that has a command-line application ready for you to use.

The AWS SDK for .NET provides the AWS Resource APIs for .NET. These resource APIs provide a resource-level programming model that enables you to write code to work more directly with resources that are managed by AWS services. A resource is a logical object that enables you to control AWS services through APIs. For example, AWS Identity and Access Management (IAM) exposes users and groups as resources that can be programmatically accessed more directly by these resource APIs than by other means.

  1. Open (double-click) the SQSConsoleApplication folder located on the desktop.

  2. Open (double-click) the SQSConsoleApplication solution file.

  3. In the How do you want to open this file? window, choose OK.

This will open the solution file in Visual Studio.

Task 4: Verify that your application is using the correct region

In this task, you need to ensure that the region code in your project (App.config) matches the region code of your deployed lab.

  1. On the right side, in the Solution Explorer pane, expand SQSConsoleApplication.

  2. Open (double-click) App.config.

This opens the App.config file in the edit pane (center screen). The region displayed in the App.config file is us-west-2. This region code was hard-coded into the application when it was developed. This is the region that the application will use when it runs. The region code in this file needs to match the region code that your lab was launched in.

  1. To the left of these instructions, look at the value of RegionCode.

This is the region that your lab was launched in.

  1. If the RegionCode value is us-west-2, you can skip to the next task.

This means that your application will use the correct region when running.

  1. In the line <add key="AWSRegion" value="us-west-2" /> change the region from us-west-2 to the value of RegionCode.

Changing the region in your application is for lab demonstration purposes only. In a production setting, you can also pass the region to each function call you make to the AWS API using the SDK.

  1. Save the change you made to App.config by pressing Control + S or by choosing the save icon .

  2. Close the App.config file.

Task 5: Resolve code issues caused by missing AWS SDK DLL references

This solution is currently only a skeleton application, and it contains errors. In this task, you will resolve the errors so that the basic console application runs by adding references to the correct AWS SDK DLLs and adding the correct using statement to the code in AWSManager.cs.

  1. In the Solution Explorer pane, open the context menu (right-click) for References, then choose Add Reference.

  2. In the Reference Manager dialog box, choose the Browse button.

  3. Browse to:

  4. Multi-select AWSSDK.Core.dll, AWSSDK.SQS.dll, and AWSSDK.SimpleNotificationService.dll, then choose Add.

  5. Choose OK.

This is how you add any of the AWS .NET DLLs to your project. The core functionality in the SDK is provided by the AWSSDK.Core.dll and it is usually required for all code using the AWS .NET SDK.

  1. In the Solution Explorer pane, open (double-click) AWSManager.cs.

  2. In the AWSManager.cs edit pane, expand Fields.

  3. Hover over the code IAmazonSQS which is underlined in red, denoting an error.

  4. In the pop-up box, choose Show potential fixes.

This error occurs because you have added the reference for the Amazon SQS DLL to your solution, but have not added the correct using statements in your AWSManager.cs file.

  1. In the drop-down list, select using Amazon.SQS;.

This will eliminate the errors.

The line using Amazon.SQS; is added near the top of AWSManager.cs. The errors should now be resolved because this line instructs the compiler to use the Amazon SQS namespace in the AWSSDK.SQS DLL you referenced earlier. You are now ready to start adding code to the application.

Add CreateQueue Code

Amazon SQS is a fast, reliable, scalable, fully managed message queuing service. Amazon SQS makes it simple and cost-effective to decouple the components of a cloud application. You can use Amazon SQS to transmit up to 2 GB of data, at any level of throughput, without losing messages or requiring other services to be always available. With Amazon SQS, you can offload the administrative burden of operating and scaling a highly available messaging cluster, while paying a low price for only what you use.

Add CreateTopic Code

Amazon SNS provides significant advantages over the complexity of developing custom messaging solutions or the expense of licensed software for systems that need to be managed and maintained on site. It runs within Amazon’s proven network infrastructure and data centers, so topics will be available whenever applications need them. To prevent messages from being lost, all messages published to Amazon SNS are stored redundantly across multiple servers and data centers. It allows applications and end users on different devices to receive notifications via mobile push notification (Apple, Google and Kindle Fire Devices), HTTP/HTTPS, Email/Email-JSON, SMS or Amazon SQS queues, or AWS Lambda functions. One common design pattern is called “fanout.” In this pattern, a message published to an SNS topic is distributed to several SQS queues in parallel. This allows you to build applications that take advantage of parallel, asynchronous processing.
